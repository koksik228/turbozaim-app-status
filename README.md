# turbozaim-app-status

Публичный remote-config (kill-switch) для iOS-приложения Turbozaim.

Приложение скачивает [`appstatus.json`](appstatus.json) при старте и при возврате из фона,
чтобы показать режим технических работ / увести пользователя на сайт, когда приложение
не может работать (поменялся бэкенд, нужно обновление и т.п.).

## Здесь НЕТ секретов

Только публичные флаги, тексты и ссылки. Никаких токенов/ключей/ПДн.

## Как пользоваться (включить «техработы»)

1. Открой `appstatus.json` → правь поле `maintenance`.
2. `active: true` — показать экран техработ; `level: "hard"` — блокирующий, `"soft"` — баннер сверху.
3. Текст/ссылку можно менять прямо здесь (в приложении они подтянутся без релиза).
4. Закончились работы → `active: false`.

Изменение доезжает до телефонов за ~5 минут (CDN-кеш GitHub).

## Поля

| Поле | Назначение |
|---|---|
| `maintenance.active` | вкл/выкл режим техработ |
| `maintenance.level` | `hard` (блок-модал) / `soft` (баннер) |
| `maintenance.title` / `message` | тексты экрана |
| `maintenance.cta` | кнопка «Перейти на сайт» (label + url; приложение открывает только `*.turbozaim.ru`) |
| `minVersion.ios` | минимальная версия приложения; ниже — экран «Обновитесь» |
| `updateUrl` | ссылка на App Store (заполнить после публикации) |
| `flags` | удалённые тумблеры фич (`registration`, `tbank`) |

Подробности решения — ADR-018 в основном репозитории приложения.
